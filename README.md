# Liora 🛡️💘

**Liora** was an MVP concept designed to challenge the superficiality of traditional dating apps. Instead of the typical "swipe culture" based purely on aesthetics, the project proposed deep connections driven by real compatibility and strict privacy.

## 🎯 The Product Vision
The core idea was to reduce noise and increase connection quality through two main pillars:

1. **Human Compatibility:** A matching engine prioritizing values and behaviors over visual metrics.
2. **Progressive Privacy:** Users held total control. Photos remained blurred by default, only being revealed after a mutual match and explicit consent, encouraging conversation before visual judgment.

## 🛠️ Engineering Milestone (My First "Architecture Leap")
This project serves as a technical record of my transition to professional Android patterns. It was here that I:
- **Moved to MVVM:** My first hands-on implementation of Model-View-ViewModel.
- **Explored Clean Architecture:** Although not yet modularized (organized by packages within the `:app` module), I applied a strict separation between Domain, Data, and UI layers.
- **Firebase Mastery:** Implemented real-time chat (Realtime DB), typing indicators/presence status, and scalable persistence with Firestore.

## 📈 Evolution & Legacy
While Liora was never released to production, it was the laboratory where I learned to think of software as a complete system (UX, Monetization, and Architecture). 

The scaling challenges I faced while organizing this project into packages were the exact trigger that led me to specialize in what I do today: **Hexagonal & Modular Architectures, and Zero-Access Security Systems.**
